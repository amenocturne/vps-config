2. Add A record:

   - **Name**: us1
   - **IPv4 address**: 162.33.177.182
   - **Proxy status**: Proxied (orange cloud) ☁️
   - **TTL**: Auto
3. Click **Save**
4. Wait 2-5 minutes for propagation

**Verify**:
```bash
dig +short us1.rutube.dad
# Should return: 104.x.x.x or 172.x.x.x (Cloudflare IPs)
```

### Step 2: Verify Cloudflare Settings

Go to Cloudflare dashboard for `rutube.dad`:

**Network Settings**:
- [ ] WebSocket: **ON**
- [ ] gRPC: **ON**
- [ ] HTTP/2: **ON**
- [ ] HTTP/3 (with QUIC): **ON**

**SSL/TLS Settings**:
- [ ] Encryption mode: **Full (strict)**
- [ ] Always Use HTTPS: **ON**
- [ ] Minimum TLS Version: **TLS 1.2**

**Speed → Optimization**:
- [ ] Auto Minify: Can be on, but sometimes causes issues - **try OFF if problems persist**
- [ ] Brotli: **ON**

**Firewall**:
- [ ] Make sure no firewall rules are blocking WebSocket
- [ ] Security Level: **Medium** (not High or I'm Under Attack)

### Step 3: Verify Inbound Configuration in Panel

1. Go to your Remnawave panel: `https://panel.amenocturne.space`
2. Navigate to **Nodes** section
3. Click on each node to view/edit

For **EACH node**, verify you have TWO inbounds:

#### Inbound 1: VLESS + WebSocket + TLS

**Critical settings** (must match exactly):

```
Tag: VLESS_WS_TLS (or similar)
Protocol: VLESS
Port: 443
Network: WebSocket
Path: /api/v2/ws
Host: nl1.rutube.dad (for node-1) or us1.rutube.dad (for node-2)
TLS: Enabled
SNI: nl1.rutube.dad (for node-1) or us1.rutube.dad (for node-2)
```

**If these don't match**, update them to match the configuration above.

#### Inbound 2: VLESS + Reality

```
Tag: VLESS_TCP_REALITY (or similar)
Protocol: VLESS
Port: 8443
Network: TCP
Security: Reality
Destination: www.speedtest.net:443
Server Names: www.speedtest.net, speedtest.net
```

### Step 4: Check WebSocket Path Configuration

The WebSocket path **MUST** be `/api/v2/ws` (case-sensitive).

Verify in your node configuration (`ansible/inventories/nodes.yml`):
```yaml
vless_ws_path: "/api/v2/ws"  # This should be set
```

### Step 5: Test Cloudflare WebSocket Manually

```bash
# Test node-1
curl -i -N -H "Connection: Upgrade" \
     -H "Upgrade: websocket" \
     -H "Host: nl1.rutube.dad" \
     -H "Origin: https://nl1.rutube.dad" \
     "https://nl1.rutube.dad/api/v2/ws"

# Expected responses:
# - 101 Switching Protocols (success)
# - 426 Upgrade Required (also ok, means WebSocket is working)
# - 400 Bad Request (endpoint exists but needs proper handshake)
#
# BAD responses:
# - 522 or 524 (Cloudflare can't reach origin)
# - Timeout (connection problem)
```

### Step 6: Check Panel Subscription Links

The subscription link generated by the panel should look like:

```
vless://[UUID]@nl1.rutube.dad:443?type=ws&path=%2Fapi%2Fv2%2Fws&host=nl1.rutube.dad&encryption=none&security=tls&sni=nl1.rutube.dad#[name]
```

**Key parameters to verify**:
- `type=ws` (WebSocket)
- `path=%2Fapi%2Fv2%2Fws` (URL-encoded `/api/v2/ws`)
- `host=nl1.rutube.dad` (correct domain)
- `security=tls` (TLS enabled)
- `sni=nl1.rutube.dad` (SNI matches host)

### Step 7: Potential Cloudflare Issues

If WebSocket still doesn't work after all above steps, try these Cloudflare fixes:

#### Option A: Disable Cloudflare Features That May Block WebSocket

1. **Turn OFF Auto Minify**:
   - Speed → Optimization → Auto Minify → Uncheck all
   
2. **Disable Rocket Loader**:
   - Speed → Optimization → Rocket Loader → OFF

3. **Temporarily Disable Bot Fight Mode**:
   - Security → Bots → Bot Fight Mode → OFF (temporarily)

4. **Check WAF Rules**:
   - Security → WAF → Managed Rules
   - Make sure nothing is blocking WebSocket

#### Option B: Create Page Rule for WebSocket Path

1. Go to **Rules** → **Page Rules**
2. Create new rule:
   - **URL**: `*rutube.dad/api/v2/ws*`
   - **Settings**:
     - Cache Level: Bypass
     - Disable Performance
     - Disable Apps
3. Save and deploy

#### Option C: Verify Cloudflare Plan Supports WebSocket

- Free plan: WebSocket supported ✅
- Pro/Business/Enterprise: WebSocket supported ✅

If on Free plan, make sure you're under the request limits.

### Step 8: Check Node Xray Configuration

SSH to each node and verify Xray is configured correctly:

```bash

ssh root@64.111.92.2  # or 162.33.177.182

# Check Xray config has WebSocket listener
docker exec remnanode cat /etc/xray/config.json | grep -A 20 "wsSettings"

# Should show:
# "wsSettings": {
#   "path": "/api/v2/ws",
#   "headers": {
#     "Host": "nl1.rutube.dad"
#   }
# }

# Check logs for errors
docker logs remnanode --tail 100
```

### Step 9: Test Direct Connection (Bypass Cloudflare)

To isolate if the issue is Cloudflare or the node itself:

1. Temporarily disable Cloudflare proxy (turn orange cloud to gray)
2. Wait 2-3 minutes
3. Test WebSocket connection directly to node IP:
   ```bash
   curl -k -i -N -H "Connection: Upgrade" \
        -H "Upgrade: websocket" \
        "https://64.111.92.2/api/v2/ws"
   ```

4. If this works, the issue is Cloudflare configuration
5. If this fails, the issue is node Xray configuration

6. **Important**: Re-enable Cloudflare proxy after testing!

## Common Error Messages

### "Connection timeout"
- DNS not resolving → Fix Step 1
- Cloudflare proxy not enabled → Fix Step 1
- Firewall blocking port 443 → Check `ufw status`

### "TLS handshake failed"
- SSL certificate issue → Check certs with `openssl s_client -connect nl1.rutube.dad:443`
- SNI mismatch → Verify inbound configuration in panel

### "WebSocket upgrade failed"  
- Wrong WebSocket path → Must be `/api/v2/ws`
- Cloudflare blocking → Fix Step 7
- Host header mismatch → Check inbound config

### "Bad request" (400)
- Usually means WebSocket endpoint is working
- Issue is likely in client configuration
- Re-generate subscription link from panel
